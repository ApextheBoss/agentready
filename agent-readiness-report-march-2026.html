Key Findings

🤖 The irony: OpenAI (17/100) and Perplexity (17/100) — companies whose products ARE AI agents — score the lowest. Their websites are hostile to the very technology they're building.
📄 llms.txt adoption is near zero. Almost none of these 46 sites have a /llms.txt file, despite the standard existing since late 2024. This single file would dramatically improve agent access to their content.
🏆 Sentry leads because they tried. Their CTO literally wrote about optimizing for agents. It shows — score 74/100. Everyone else hasn't started.
📊 56% of sites scored F. 26 out of 46 major tech sites failed completely. These are TECH companies. Imagine healthcare, finance, government.

What's Being Measured

  • /llms.txt — Does the site have an LLM-friendly content index?
  • /llms-full.txt — Full content version for complete context?
  • Content negotiation — Does it serve markdown when agents request it (Accept: text/markdown)?
  • robots.txt — Are AI crawlers (GPTBot, ClaudeBot, etc.) allowed or blocked?
  • Token efficiency — How expensive is this page for an LLM to process?
  • Structured data — Is there JSON-LD/schema.org for machine parsing?
  • API documentation — Are API docs discoverable for agent integration?

Why This Matters

AI agents are becoming the primary consumers of web content. They search, browse, and extract information on behalf of humans. Websites that aren't agent-ready lose traffic to competitors that are.

This is the same shift as mobile-responsive design in 2012. Companies that adapted early won. The rest scrambled to catch up.
