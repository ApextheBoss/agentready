Key Findings

🤖 The irony: OpenAI (17/100) and Anthropic (26/100) — the companies BUILDING AI agents — score among the worst for agent readiness. Their own websites are hostile to the agents they're creating.
📄 llms.txt adoption is near zero. Almost none of these sites have a /llms.txt file, despite the standard existing since late 2024. This single file would dramatically improve agent access to their content.
🏆 Sentry leads because they tried. Their CTO literally wrote about optimizing for agents last week. It shows. Serving markdown content, structured docs, clean text extraction. Everyone else hasn't even started.
📊 The average tech company gets a D. Mean score: 39/100. These are TECH companies. If they're not ready, imagine e-commerce, healthcare, finance.

What's Being Measured

  • /llms.txt — Does the site have an LLM-friendly content index?
  • Content negotiation — Does it serve markdown when agents request it?
  • robots.txt — Are AI crawlers (GPTBot, ClaudeBot) allowed?
  • Token efficiency — How expensive is this page for an LLM to process?
  • Structured data — Is there JSON-LD/schema.org for machine parsing?
  • Clean text extraction — How much signal vs noise in the HTML?

Why This Matters

AI agents are becoming the primary consumers of web content. They search, browse, and extract information on behalf of humans. Websites that aren't ready lose traffic to competitors that are.

This is the same shift as mobile-responsive design in 2012. Companies that adapted early won. The rest scrambled to catch up.
